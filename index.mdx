## Why Xinity

Cloud AI assumes your data can leave the building. For regulated enterprises — media companies, manufacturers, banks, public institutions — it can't. Xinity gives them a complete AI platform that runs entirely on their own hardware.

## Get started in minutes

<Steps>
  <Step title="Install the Xinity CLI">
    ```bash
    curl -fsSL https://github.com/xinity-ai/xinity-ai/releases/latest/download/install.sh | bash
    ```
  </Step>
  <Step title="Deploy the full stack">
    ```bash
    xinity up all
    ```
    This installs the gateway, dashboard, and database as systemd services on your server.
  </Step>
  <Step title="Create your admin account">
    ```bash
    xinity configure dashboardUrl http://localhost:3100
    xinity act onboarding.cli
    ```
  </Step>
  <Step title="Deploy a model and start calling the API">
    ```bash
    xinity act deployment.create '{"name": "Phi-3 Mini", "publicSpecifier": "phi-3-mini", "modelSpecifier": "phi3:mini", "enabled": true}'
    curl http://localhost:3000/v1/chat/completions \
      -H "Authorization: Bearer sk_..." \
      -H "Content-Type: application/json" \
      -d '{"model": "phi-3-mini", "messages": [{"role": "user", "content": "Hello from on-prem."}]}'
    ```
  </Step>
</Steps>

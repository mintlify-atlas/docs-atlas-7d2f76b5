---
title: "Get started with Xinity AI: deploy your first model"
description: "Install the CLI, deploy the stack, and make your first OpenAI-compatible API call to your own on-premises model in under 10 minutes."
---

This guide walks you through installing Xinity AI on a Linux server, deploying a model, and calling it using the OpenAI-compatible API. The whole flow takes less than 10 minutes on a fresh machine.

<Note>
  **Prerequisites:** Xinity requires a Linux server with systemd. For Docker-based environments (macOS, Windows, cloud VMs without systemd), use the [Docker Compose deployment](/deployment/docker) instead.
</Note>

<Steps>
  <Step title="Install the CLI">
    Download and install the Xinity CLI with the install script. It detects your platform (x64 or arm64), downloads the matching binary, verifies the SHA256 checksum, and installs it to `~/.local/bin`.

    ```bash
    curl -fsSL https://get.xinity.ai/install.sh | bash
    ```

    Verify the install:

    ```bash
    xinity --version
    ```
  </Step>

  <Step title="Deploy the stack">
    Run `xinity up all` to install the gateway, dashboard, and database as systemd services. The command downloads each component binary from GitHub Releases, prompts you for configuration, generates systemd unit files, and starts the services.

    ```bash
    xinity up all
    ```

    This sets up:
    - **Gateway** — the OpenAI-compatible API proxy (default port 3000)
    - **Dashboard** — the management UI and REST API (default port 3100)
    - **Database** — PostgreSQL with Redis for caching and load balancer state

    To install on a remote server instead, add the `--target-host` flag:

    ```bash
    xinity up all --target-host user@your-server
    ```

    Check that all services are healthy after installation:

    ```bash
    xinity doctor
    ```
  </Step>

  <Step title="Create your admin account">
    Point the CLI at your dashboard and run the onboarding flow. This creates your first admin account without needing a browser.

    ```bash
    xinity configure dashboardUrl http://localhost:3100
    xinity act onboarding.cli
    ```

    The onboarding command walks you through creating an admin user and your first organization interactively.
  </Step>

  <Step title="Deploy a model">
    Create a model deployment. This example uses Phi-3 Mini, a lightweight model that downloads quickly.

    ```bash
    xinity act deployment.create '{"name": "Phi-3 Mini", "publicSpecifier": "phi-3-mini", "modelSpecifier": "phi3:mini", "enabled": true}'
    ```

    The deployment is written to the database. The daemon running on your inference node detects it and begins pulling the model from the Ollama registry.
  </Step>

  <Step title="Wait for the model to be ready">
    Poll the deployment list until the status shows `ready`. Model download time depends on your connection speed and model size.

    ```bash
    xinity act deployment.list '{"withStatus": true}'
    ```

    You'll see a `status` field transition through `downloading` → `installing` → `ready`. Once the status is `ready`, the model is available for inference.
  </Step>

  <Step title="Call the API">
    Make an inference request to the OpenAI-compatible gateway endpoint:

    ```bash
    curl http://localhost:3000/v1/chat/completions \
      -H "Authorization: Bearer sk_..." \
      -H "Content-Type: application/json" \
      -d '{
        "model": "phi-3-mini",
        "messages": [{"role": "user", "content": "Hello from on-prem."}]
      }'
    ```

    Replace `sk_...` with an API key from the dashboard (**Settings → API Keys**).

    You can also use the Python OpenAI SDK — just change the `base_url`:

    ```python
    from openai import OpenAI

    client = OpenAI(
        base_url="http://localhost:3000/v1",
        api_key="sk_..."
    )

    response = client.chat.completions.create(
        model="phi-3-mini",
        messages=[{"role": "user", "content": "Hello from on-prem."}]
    )

    print(response.choices[0].message.content)
    ```

    Any existing OpenAI SDK code works unchanged — just point `base_url` at your Xinity instance.
  </Step>
</Steps>

<Tip>
  For production deployments, multi-node GPU setups, Docker Compose, or NixOS, see the [deployment overview](/deployment/overview).
</Tip>
